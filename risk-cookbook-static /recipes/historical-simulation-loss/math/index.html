Recipe 03 - Math

Historical Simulation Loss

The mathematics behind the historical simulation loss recipe.

Back to recipe

Model setup

Historical simulation does not start with a parametric distribution like \(\mathcal{N}(\mu, \Sigma)\) or \(t_\nu(\mu, \Sigma)\). Instead, it takes a matrix of observed return scenarios

\[ R_1, R_2, \dots, R_T \in \mathbb{R}^n \]

and treats those scenarios as the empirical loss distribution for the current portfolio.

From return history to loss history

For each historical return vector, the page computes the corresponding portfolio loss:

\[ L_t = -w^\top R_t, \qquad t = 1, \dots, T \]

The collection \(L_1, \dots, L_T\) is the empirical loss sample.

Empirical VaR and Expected Shortfall

Once the loss sample is built, the page sorts it and reads off the \(\alpha\)-quantile:

\[ \widehat{\mathrm{VaR}}_{\alpha}(L) = \text{empirical quantile of } \{L_t\}_{t=1}^T \]

The empirical expected shortfall is the average of all losses beyond that threshold:

\[ \widehat{\mathrm{ES}}_{\alpha}(L) = \frac{1}{N_\alpha} \sum_{t: L_t \ge \widehat{\mathrm{VaR}}_{\alpha}(L)} L_t \]

where \(N_\alpha\) is the number of losses in the tail.

What the lookback window means

If the page uses only the most recent \(m\) observations, then the empirical distribution changes to

\[ \{L_{T-m+1}, \dots, L_T\} \]

This is why the lookback sensitivity chart matters: historical simulation is entirely dependent on which past scenarios are included.

What the simulation is doing

Strictly speaking, this page is more of a historical scenario replay than a simulation. The histogram is empirical, the worst-scenarios table identifies the largest realized losses, and the cumulative path view compounds the user-provided return history into a portfolio value series.

The strength of historical simulation is that it is intuitive and model-light. Its weakness is that it cannot produce scenarios that never occurred in the input sample.
