Recipe 04 - Math

Factor Model Loss

The mathematics behind the factor model loss recipe.

Back to recipe

Model setup

The factor model assumes that each asset return can be decomposed into a common-factor piece and an idiosyncratic residual:

\[ R = Bf + \varepsilon \]

Here \(B\) is the exposure matrix, \(f\) is the vector of factor returns, and \(\varepsilon\) is the vector of asset-specific shocks.

Factor covariance structure

The page assumes Gaussian factor shocks with

\[ f \sim \mathcal{N}(\mu_f, \Sigma_f) \]

and diagonal specific risk

\[ \varepsilon \sim \mathcal{N}(0, D) \]

where \(D\) is diagonal. This implies the asset covariance matrix

\[ \Sigma_R = B\Sigma_f B^\top + D \]

From factors to portfolio loss

If the portfolio has weights \(w\), then the portfolio loss is

\[ L = -w^\top R = -w^\top Bf - w^\top \varepsilon \]

This expression makes the decomposition transparent: one term comes from systematic factor moves, and one term comes from specific shocks.

Risk decomposition

The page computes the total portfolio variance

\[ \mathrm{Var}(L) = w^\top (B\Sigma_f B^\top + D) w \]

and splits it into

\[ \mathrm{Systematic\ risk} = w^\top B\Sigma_f B^\top w \]
\[ \mathrm{Specific\ risk} = w^\top D w \]

This is the mathematical reason the main page can show factor contribution charts and systematic-vs-specific shares.

What the simulation is doing

The simulation draws factor returns \(f^{(m)}\), draws specific shocks \(\varepsilon^{(m)}\), builds asset returns \(R^{(m)}\), and then converts those draws into portfolio losses \(L^{(m)}\). The path panel compounds repeated factor shocks through time so the user can see how common-factor exposure shapes portfolio outcomes.

This page changes the geometry of the model. Instead of specifying risk directly in asset space, it builds risk through a lower-dimensional factor structure.
